# Contributing to Frequency of Life  
  
Thank you for your interest in contributing to Frequency of Life.  
  
This project is designed to encourage collaboration **without exposing proprietary core logic**.  
  
---  
  
## Contribution Areas (Open)  
  
You may contribute to:  
- UI / frontend components  
- SDKs and API clients  
- Plugins and integrations  
- Documentation  
- Developer tooling  
- Tests and examples  
- Bug fixes in open repositories  
  
You may **not** contribute to:  
- Core algorithms  
- Scoring or weighting logic  
- ML models or training data  
- Internal optimization strategies  
  
---  
  
## How to Contribute  
  
1. Fork the repository  
2. Create a feature branch  
3. Make focused, well-documented changes  
4. Submit a Pull Request (PR)  
5. Address review feedback

## CLA Requirement  
  
All contributors must agree to the **Contributor License Agreement (CLA)**.  
By submitting a PR, you confirm agreement with the CLA.  
  
---  
  
## Design Philosophy  
  
- Interfaces over internals  
- Clear contracts  
- No reverse engineering  
- Respect for boundaries  
  
---  
  
## Communication  
  
- Use GitHub Issues for bugs and features  
- Use GitHub Discussions for ideas and questions  
- Be respectful and constructive  
  
Thank you for helping build Frequency of Life.
